1.What is the Difference among Map, filter, foreach and find?

Map: A new array with the same number of items as the original array is produced by applying a specified function to each element of an array using the map function. The initial array is not changed by it.

Filter: The filter function accepts two inputs: an array and a callback function. It applies the callback function to each element of the array and outputs a new array that only contains elements for which the callback function returns true. The initial array is unaltered.

forEach: The forEach function applies a given function to each element of an array but does not return anything. It is used when we want to iterate through an array and perform some action on each element, but we don't need to create a new array.

Find: The find function accepts an array and a callback function as input, applies the callback function to each element of the array, and outputs the first element for which the callback function returns true. The condition returns undefined if no elements match it.

3. What is event loop and how it works?

JavaScript's concurrency paradigm, known as the event loop, enables asynchronous code to run without blocking. JavaScript uses single-threaded execution, which allows for the simultaneous execution of just one piece of code. The execution of asynchronous code is, however, made possible by a number of JavaScript features, including callbacks, promises, and async. When an asynchronous action is launched, JavaScript doesn't wait for it to finish; instead, it immediately starts running other code. JavaScript's event loop is in charge of controlling the execution of asynchronous code. It continuously monitors the message queue and the call stack to see whether any new code needs to be run.

Thats way the event loop works:

1. The JavaScript engine executes the code synchronously, adding functions to the call stack.

2. When an asynchronous operation is started, it is removed from the call stack, and its callback function is added to the message queue.

3. When the call stack is empty, the event loop checks the message queue for pending tasks.

4. If there are pending tasks in the message queue, the event loop picks the first task and adds its callback function to the call stack for execution.

5. The JavaScript engine executes the callback function, and any synchronous code in the function is added to the call stack.

6. The event loop repeats the process, checking the call stack and message queue for new tasks.
